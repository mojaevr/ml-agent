## Project Overview

ML Agent system that uses LLM agents (DeepSeek/Qwen via OpenRouter) to autonomously and iteratively optimize binary classification (ROC-AUC). The agents generate Python code for model training, which runs in a sandboxed subprocess, gets validated independently, and results feed back into the next iteration.

Language: Russian (all UI messages, comments, and prompts are in Russian).

## Commands

```bash
# Install dependencies
pip install -r requirements.txt

# Prepare data (split train_full.csv → train.csv + val.csv, analyze schema, generate runner)
python orchestrator.py --prepare_data

# Run agent optimization (main workflow)
python orchestrator.py --iterations 10 --agents 2 --budget 3.0 --model deepseek/deepseek-r1

# List all runs sorted by ROC-AUC
python orchestrator.py --list

# Generate Kaggle submission from best run
python orchestrator.py --submit

# Generate from specific run
python orchestrator.py --submit --run_id run_20240101_143022 --output my_submission.csv

# Skip inter-agent discussion phase
python orchestrator.py --iterations 5 --agents 1 --no-discussion
```

No test suite exists. Validation is done via `validator.py` computing ROC-AUC on held-out `data/val.csv`.

## Architecture

### Two Parallel Architectures for Agent Code

The system has **two code generation paths** that coexist — understanding this duality is critical:

1. **Legacy path** (currently active in `agent_loop.py`'s `_build_system()`): Agent writes `train_code.py` containing `build_model(X_train, y_train, X_val, y_val)` and `build_features(X_raw)` operating on numpy arrays. The template uses `exec(open(os.environ['AGENT_RUNNER_PATH']).read())` to run `data/runner.py` (generated by `data_prep.py`), which handles data loading, encoding, and splitting.

2. **New path** (`runner.py`, `ml_dataset.py`, `agent_prompt.py`, `preflight.py`): Agent writes `train_code.py` with `build_features(dataset)` and `build_model(dataset)` operating on an `MLDataset` object. The standalone `runner.py` at project root orchestrates loading, FE, training, validation, and artifact saving.

The legacy path is what `orchestrator.py` → `agent_loop.py` → `sandbox_runner.py` actually uses. The new path (`runner.py` at root, `ml_dataset.py`, `agent_prompt.py`, `preflight.py`) appears to be a refactoring in progress.

### Execution Flow (one iteration)

```
orchestrator.py
  → agent_loop.propose_approach()     # LLM proposes idea (parallel per agent)
  → agent_loop.discuss_approaches()   # LLM agents discuss (parallel)
  → agent_loop.generate_training_code()  # LLM generates train_code.py
  → agent_loop.preflight_check()      # AST-based static analysis
  → sandbox_runner.run_training_code() # subprocess with env vars, 8min timeout
  → [auto-fix loop: pip install / LLM fix, up to 3 attempts]
  → validator.compute_roc_auc()       # independent ROC-AUC on val_preds.npy
  → agent_loop.analyze_results()      # LLM analyzes what went wrong/right
  → memory.add_attempt()              # persist to memory/attempts_log.json
```

### Key Modules

- **orchestrator.py** — Entry point. CLI parsing, parallel round execution via `ThreadPoolExecutor`, Kaggle submission generation.
- **agent_loop.py** — All LLM interaction. Contains system prompts, code generation, error fixing, approach proposal/discussion, result analysis. Calls OpenRouter API directly via `requests`.
- **sandbox_runner.py** — Runs agent code in isolated subprocess. Sets env vars (`AGENT_RUN_DIR`, `AGENT_TRAIN_PATH`, `AGENT_RUNNER_PATH`, `AGENT_MODE`). 8-minute timeout.
- **validator.py** — Independent ROC-AUC computation. Agent cannot access `val.csv` directly.
- **memory.py** — JSON-based attempt history (`memory/attempts_log.json`). Provides summaries to agents so they avoid repeating failed approaches.
- **budget_tracker.py** — Tracks daily OpenRouter token spending. Hard stops when daily budget exceeded.
- **data_prep.py** — Splits `train_full.csv` → `train.csv`/`val.csv`, analyzes schema, generates `data/runner.py` (the encoding/splitting boilerplate that agent code exec's).
- **ml_dataset.py** — `MLDataset` class with sklearn pipeline integration. Used by the new `runner.py` path.
- **preflight.py** — Static checks for the new-path `train_code.py` (checks for `build_features`/`build_model` with dataset arg).
- **agent_prompt.py** — System prompt for the new-path agent interface.
- **train_code.py** — Baseline template showing the new-path agent interface.

### Data Flow

- `data/train_full.csv` — raw input (user provides)
- `data/train.csv` / `data/val.csv` — 80/20 stratified split (generated once, fixed seed 42)
- `data/test.csv` — test set for Kaggle submission (user provides)
- `data/data_schema.json` — auto-generated column statistics
- `data/runner.py` — auto-generated boilerplate (encoding + split logic, exec'd by agent code)
- `runs/<run_id>/` — per-iteration artifacts: `train_code.py`, `predict_code.py`, `model.pkl`, `val_preds.npy`, `meta.json`

### Security Boundary

The agent LLM cannot access `val.csv`. The sandbox sets only `AGENT_TRAIN_PATH` (pointing to `train_full.csv`). Validation is done externally by `validator.py` reading `val_preds.npy` produced by the agent and comparing against the held-out `val.csv`.

### Environment Variable Contract (sandbox)

| Variable | Description |
|---|---|
| `AGENT_RUN_DIR` | Absolute path to run directory |
| `AGENT_TRAIN_PATH` | Path to `train_full.csv` |
| `AGENT_RUNNER_PATH` | Path to `data/runner.py` |
| `AGENT_MODE` | `train` or `predict` |
| `AGENT_TEST_PATH` | Path to `test.csv` (predict only) |
| `OPENROUTER_API_KEY` | API key (set by user in environment) |

### LLM Models

Configured via `--model` flag. Prices tracked in `budget_tracker.py:MODEL_PRICES`. Default: `deepseek/deepseek-r1`.
